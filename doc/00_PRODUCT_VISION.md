# WorkMemory 00: 产品愿景与核心价值 (Product Vision & Core Values)

> **文档定位**：本项目最高精神纲领，定义“为什么做”以及产品的灵魂边界。AI Coding Agent 在设计任何交互、编写任何文案或设计 UI 时，必须以此为最高准则。

---

## 1. 产品一句话定义

**WorkMemory 今日记忆** 是一个运行在 Windows 平台上的**本地优先、常驻桌面的个人工作记忆伙伴**。它在后台安静、低打扰地整理你每天的屏幕碎片，将它们沉淀为可复看、可检索、可串联、可沉淀的个人知识系统，并通过一个温和、有情绪、有状态的桌面伙伴（Mascot）陪伴你度过每一个高效工作日。

---

## 2. 核心价值矩阵：颜值、好玩、有趣、实用、有用

WorkMemory 拒绝做成冰冷的“企业级监控后台”或单纯的“时间审计/番茄钟工具”。我们将产品的生命力归结为以下五个核心维度：

```
       ┌───────────────────────────┐
       │     WorkMemory 核心价值    │
       └──────────────┬────────────┘
         ┌────────────┼────────────┐
   【情感与感官】                 【功能与资产】
  ┌──────┴──────┐              ┌──────┴──────┐
  │  1. 颜值 (Aesthetic)       │  4. 实用 (Practical)
  │  2. 好玩 (Playful)         │  5. 有用 (Valuable)
  │  3. 有趣 (Interesting)     │
  └─────────────┘              └─────────────┘
```

### 2.1 颜值 (Aesthetic) —— 第一生产力
*   **愿景**：颜值即正义。作为一个常驻桌面、用户每天都要看上百次的软件，视觉的舒适度和分寸感至关重要。
*   **设计原则**：干净、克制、轻材质、温和。使用细腻的动效表达状态，不使用大面积饱和色，不产生视觉压迫感。
*   **交付标准**：主窗口的任意一个主页面，截出一张图来，都必须是一张好看的产品宣发图。如果页面“功能完备但不好看”，默认视作未完成。

### 2.2 好玩 (Playful) —— 桌面伙伴的情绪价值
*   **愿景**：让“本地记录”这件略显枯燥（甚至带有一丝隐私焦虑）的事变得温馨和令人期待。
*   **机制**：引入桌面伙伴（Mascot），它拥有“拉帘闭眼（隐私状态）”、“戴上耳机（深度专注）”、“打瞌睡（电脑空闲）”、“递便签（发现灵感或报告就绪）”等丰富的微动态。
*   **体验**：它不是强行跳出来说话的“微软回形针（Clippy）”，而是一个在你屏幕边缘安静陪伴的、有生命力的小摆件。

### 2.3 有趣 (Interesting) —— 拒绝教条与审判
*   **愿景**：系统不扮演“班主任”或“监工”的角色，不对用户的状态进行任何道德批判。
*   **话术设计**：不用“你今天太低效了”或“你摸鱼了 2 小时”，而是使用“今天在非工作应用停留了一阵，累了就歇会儿，需要 10 分钟后叫你吗？”或者“今天沟通挺碎的，我帮你把线索合起来？”
*   **情感关联**：将枯燥的“应用时长统计”翻译成“今日故事（Daily Story）”，让用户像翻看每日手记一样回顾自己的一天。

### 2.4 实用 (Practical) —— 零操作捕获与闪电汇报
*   **愿景**：解决“今天日报怎么写”、“我上周到底忙了啥”的刚性痛点。
*   **痛点攻克**：
    *   **零操作**：全后台自动感知，无需手动打点，无需记录开始/结束。
    *   **闪电生成**：下班前一键勾选关键事件，AI 蒸馏生成排版优美、直抄日报系统的富文本，直接复制，下班无忧。
    *   **Ghost Capture（快速捕捉）**：按快捷键 `Ctrl+Shift+C` 即可将屏幕任意区域的文字抓成灵感卡，直接存入 Wiki 或今日线索。

### 2.5 有用 (Valuable) —— 从“碎片日志”到“认知资产”
*   **愿景**：产品越用越聪明，时间跨度越长越有价值。
*   **资产转化**：
    *   **Segment 级（碎片像素）** → **Episode 级（人类理解的事件）**：去除重复截图与 OCR 噪音，将一天聚合成 3-5 个清晰的主线故事。
    *   **Wiki 知识库**：一键将高价值片段沉淀为 Wiki 页面，通过 `[[wikilink]]` 双链和图谱建立人、事、项目、时间的连接，在年底绩效回顾或复盘时发挥巨大价值。

## 3. 核心产品观：记忆不是日志，而是可回到现场的线索网

*   **传统记录工具的误区**：把时间切成碎块（10:00 飞书、10:15 浏览器、10:30 Word），给用户一堆无序、冰冷且充满噪音的 Log。
*   **WorkMemory 的解法**：记忆是一张网。它由**人（People）、事（Projects/Tasks）、时间（Dates/Epochs）、文档（Documents/Urls）**这三根轴交织而成。
*   用户搜索“退款状态字段”，系统不应该只给一个搜索结果，而是将“阅读需求 → 和开发沟通字段 → 接口验证 → 记录日报 → 沉淀 Wiki”的完整历史线索像电影胶片一样串联起来。

---

## 4. 产品的边界：绝对禁止做的事 (Anti-Patterns)

为了保护用户绝对的安全感，以下行为属于**红线禁止**。AI Coding Agent 不得在任何情况下编写、绕过或尝试实现这些逻辑：

1.  **禁止网络上传原始截图**：用户的截图必须在本地处理。除非用户明确授权上传某段“纯文本”给大模型（且大模型 API 路径由用户直接控制），任何截图和原始像素信息不得离域。
2.  **禁止键盘记录器 (Keylogger)**：系统绝不监听用户击键的键值。我们只感知前台窗口和窗口标题。
3.  **禁止强弹窗打扰**：除非用户手动设置了特定的番茄钟/提醒，系统绝对不允许发送强遮挡的 Modal 或 Native Toast。所有的主动发现和提醒必须首先进入“洞察中心（Insights）”，或通过桌面伙伴头顶轻柔、可一键关闭的气泡表达。
4.  **禁止暗自后台联网分析**：系统后台处理（OCR、降噪、Episode 聚合）全部在本地 CPU 运行。只有在整点蒸馏或用户主动要求生成日报、且配置了 OpenAI API Key 时，才发起出境的 HTTPS API 请求。
5.  **禁止评判式、指责性话术**：对话、气泡、空状态、卡片文案中，禁止出现任何暗示用户“不努力”、“浪费时间”、“效率低下”的字眼。
