# WorkMemory 05: 交互逻辑、桌面伙伴与主动智能规范 (Interaction Logic)

> **文档定位**：定义用户与常驻产品的相处模式、桌面伙伴（Mascot）的完整互动状态机、系统何时克制、何时以极低干扰主动，以及无 AI 降级模式下的交互流流转。AI Coding Agent 在设计组件状态机、全局定时任务、右键托盘/Mascot 菜单、弹出气泡逻辑时，必须严格遵守本文档。

## 1. 记录状态机 (Recorder State Machine)

系统后台捕获与状态机具有绝对严密的逻辑，任何状态流转都会通过 IPC 广播给主窗口和 Mascot 窗口：

```
       ┌───────────────────────┐
       │     Initializing      │
       └───────────┬───────────┘
                   │
                   ▼
  ┌────────► 🟢 Recording ◄──────────┐
  │            │      ▲              │
  │            │      │              │
(Idle 检测)  (恢复) (暂停)         (恢复)
  │            │      │              │
  │            ▼      │              │
  │          🟡 Idle ─┘              │
  │                                  │
  └────────► 🔴 Paused ◄─────────────┤
               │      ▲              │
            (隐私)  (恢复)          (恢复)
               │      │              │
               ▼      │              │
          🟣 PrivacyMode ────────────┘
```

### 1.1 状态转移矩阵与表现规范

| 源状态 | 触发条件 | 目标状态 | 托盘图标颜色 | Mascot 动画（Spritesheet Row） | 主窗口记录状态条文字 |
|---|---|---|---|---|---|
| **Recording** | 3分钟无键盘鼠标输入 | **Idle** | 柔和绿闪烁 | `sleep`（Row 3）打瞌睡 Zzz | 电脑处于闲置状态 |
| **Recording** | 命中隐私黑名单规则 | **PrivacyMode** | 紫色锁定 | `special`（Row 8）拉帘/闭眼 | 🔒 隐私保护中 |
| **Recording** | 用户手动点击"暂停" | **Paused** | 灰色静止 | `sit`（Row 4）静止发呆/抱臂 | 记录已暂停 |
| **Paused** | 用户手动点击"恢复" | **Recording** | 亮绿色 | `idle`（Row 0）轻微呼吸/握笔 | ● 正在记录中... |
| **PrivacyMode** | 用户手动切换为普通模式 | **Recording** | 亮绿色 | `idle`（Row 0）轻微呼吸/握笔 | ● 正在记录中... |

> **实现说明**：Mascot 动画状态切换由前端 `recorderStateToMascotState()` 函数统一处理（定义见 `04_UI_SPEC.md` 第 5.5 节），前端监听 IPC 事件 `recorder-state-changed` 后自动更新 `MascotSprite` 的 `state` prop。

## 2. 桌面伙伴 (Mascot) 互动机制

桌面伙伴（Mascot）是 WorkMemory 的人格化代理，必须遵循**"极低打扰、安静陪伴、右键即达"**的原则。

### 2.1 Mascot 窗口行为硬约束
1.  **绝不遮挡专注区域**：当检测到前台窗口处于全屏、演示模式、视频播放状态，或检测到特定的专业 IDE/设计软件（如 `devenv.exe`, `idea64.exe`, `Photoshop.exe`）在前台时，Mascot 窗口必须**自动降低透明度至 0.15**，甚至在 Settings 中可配置自动隐身（Hide when focused）。
2.  **绝对禁止主动夺焦**：Mascot 窗口必须设置 `skip_taskbar = true`，且其底层窗口句柄必须启用 `WS_EX_NOACTIVATE` 属性，确保用户点击 Mascot 时不会导致当前编辑的应用失去焦点。
3.  **吸附停靠**：支持鼠标左键按住拖拽，松开时自动平滑磁吸停靠在屏幕最右侧或右下角贴边位置。拖拽中播放 `drag`（Row 7）动画，松手后播放 `fall`（Row 6）一次性落地动画，落地后恢复正常状态动画。

### 2.2 Mascot 完整状态行为对照表

以下是所有 Spritesheet 行与用户可感知场景的完整对照关系，AI Coding Agent 必须按此实现：

| Spritesheet Row | 动画名 | 触发场景 | 循环 |
|---|---|---|---|
| Row 0 `idle` | 轻微呼吸/握笔待机 | 系统正在记录（默认常态） | ✅ |
| Row 1 `walk` | 走路 | 拖拽伙伴移动过渡（连接 drag 和 fall） | ✅ |
| Row 2 `run` | 快速奔跑 | 伙伴贴边磁吸时的高速移动 | ✅ |
| Row 3 `sleep` | 打盹 Zzz | 系统 Idle（用户离开电脑3分钟） | ✅ |
| Row 4 `sit` | 端坐发呆 | 用户手动暂停记录 | ✅ |
| Row 5 `jump` | 跳跃欢庆 | 日报生成完毕（一次性庆祝） | ❌ 播完恢复 |
| Row 6 `fall` | 落地缓冲 | 拖拽松手落地过渡 | ❌ 播完恢复 |
| Row 7 `drag` | 被拖拽 | 用户正在拖动伙伴移动位置 | ✅ |
| Row 8 `special` | 拉帘/闭眼 | 隐私模式激活（PrivacyMode） | ✅ |

### 2.3 情绪气泡控制算法 (Bubble Interruption Cap)
为防止 Mascot 变成招人烦的垃圾弹窗，系统必须在内存中执行以下控制规则：
*   **每小时上限**：任何情况下，Mascot 每小时主动弹出气泡次数**不得超过 1 次**。
*   **隐私拦截提示限制**：命中隐私规则自动"拉帘"时，仅在第一次闪烁紫色提示，连续命中时不重复弹气泡。
*   **气泡自动淡出**：所有主动气泡在浮现 **6 秒**后必须自动淡出消失。每个气泡右上角必须带有一个微小的"×"关闭按钮。如果用户点击了"×"，**当天 Mascot 不得再针对该类别主动推送任何气泡**。
*   **连续关闭降频**：用户当天累计关闭气泡 3 次后，当天不再主动推送任何气泡。

用户右键点击 Mascot 弹出菜单，菜单项：

## 3. 主动智能时机与话术规范 (Insights Timing)

系统的主动交互必须温和且有边界。以下是系统允许触发主动的三个场景及严苛的话术约束：

### 3.1 每日下班复盘 (Daily Story Wrap)
*   **触发时机**：系统检测到系统时钟处于 `17:30 - 19:30` 之间，且今日有效记录时长（`duration_seconds`）累计超过 3 小时。
*   **Mascot 表现**：切换至 `jump`（Row 5）一次性跳跃动画，跳跃落地后弹出气泡。
*   **气泡文案**："*今天的记忆已经整理好了，要现在生成日报吗？*"
*   **话术禁止**：严禁出现暗示工作时长不够、或是评价用户效率低的话术（如："你今天只工作了3小时"）。
*   **推荐话术**：一律使用中性叙述："*今天一共捕获了 5 条工作线索，主要关于 [项目A]，点击一键生成日报素材，轻松下班。*"

### 3.2 深度专注反馈与休息提醒 (Focus & Break)
*   **触发时机**：当用户在单一工作进程或同一个 Episode 主题下持续专注超过 **45 分钟**，且期间未发生频繁的窗口切换。
*   **Mascot 表现**：保持 `idle`（Row 0）动画，在第 45 分钟时弹出气泡，伙伴继续安静陪伴。
*   **气泡文案**："*专注很久了，小记建议你站起来活动一下，喝杯水。需要我 10 分钟后提醒你回来吗？*"

### 3.3 焦虑/碎片切换降噪建议 (Anxiety/Interruption Filter)
*   **触发时机**：系统检测到用户在 10 分钟内高频切换前台窗口超过 30 次，且多为不同类别的 IM、网页、文档。
*   **降噪表现**：系统在 InsightsView 生成一张轻量级建议卡，Mascot 头顶轻提示："*刚刚的信息流有些碎，需要我帮你把这些临时沟通片段合并成一条'日常事务整理'，方便稍后写日报吗？*"

## 4. 无 AI 降级模式下的交互流 (Offline UI Adaptation)

当用户未填 API Key 或网络离线时，前端 UI 必须发生相应的交互自适应流转，确保"实用"和"好玩"不打折：

### 4.1 今日页总结卡 (SummaryBar)
*   **有 Key 状态**：展示 AI 提炼的高级叙事总结，点击可直接让 AI 修改。
*   **降级状态**：
    *   卡片标题显示为 `📊 今日活动线索统计`。
    *   展示纯本地统计的 Bullet 项（例如："*今日电脑运行了 4 小时 20 分。活跃时间集中在 10:00 - 12:00 (VS Code) 和 14:00 - 15:30 (Edge)。*"）。

### 4.2 日报一键生成 (Daily Report Generation)
*   **有 Key 状态**：用户点击"生成日报"，系统进入 AI 实时生成流，富文本编辑器流式打字输出排版优雅的日报。
*   **降级状态**：
    *   用户点击"生成日报"，系统**免调用 API，直接零延迟渲染**基于精美模板拼装的 Markdown Bullet。
    *   **拼接模板规范**：
        ```markdown
        # 今日工作复盘 (WorkMemory 自动整理)
        日期: [YYYY-MM-DD]

        ## 💻 活跃线索流 (按持续时间排序)
        *   **[10:00 - 11:20] 推进订单退款字段确认**
            - 活跃应用: Edge, Word
            - 关键标题: "订单系统字段文档", "飞书沟通群"
        *   **[14:00 - 15:15] 测试验证与 Debug**
            - 活跃应用: VS Code, Chrome
            - 关键标题: "refund_test.go", "Localhost:3000"

        *💡 提示: 已启用本地降级模板。填入 OpenAI Key 后即可享受 AI 自动润色与逻辑合并。*
        ```
